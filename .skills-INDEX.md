## 🚀 QUY TRÌNH THỰC THI (SOP)

1. **Scout**: `/scout` để nắm rõ các file bị ảnh hưởng.
2. **Plan**: `/ck:plan <task>` → Tạo plan tại `.skills/plans/`.
3. **Cook**: `/ck:cook` → Thực thi code từng phase.
4. **Test**: `/ck:test` → Đảm bảo unit test coverage > 80%.
5. **Review**: `/review` → Đối chiếu với `rules.md`.
6. **Commit**: `/git:cm` → Theo chuẩn Conventional Commits.
